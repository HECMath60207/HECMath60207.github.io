Qu’est-ce que l’apprentissage automatique ?

L’apprentissage automatique est une application de l’intelligence artificielle (IA) qui offre aux systèmes la capacité d’apprendre et de s’améliorer automatiquement à partir de l’expérience sans être explicitement programmés. L’apprentissage automatique se concentre sur le développement de programmes informatiques qui peuvent accéder aux données et les utiliser pour apprendre par eux-mêmes.

Approches de l’apprentissage automatique

  • Apprentissage supervisé : L’ordinateur est présenté avec exemple entrées et leurs sorties souhaitées (pensez OLS).
  • Apprentissage non supervisé : Aucune étiquette n’est donnée à l’apprentissage algorithme, le laissant seul pour trouver une structure dans son entrée.
  • Apprentissage par renforcement : Un programme informatique interagit avec un environnement dynamique dans lequel il doit atteindre un certain objectif. Comme il navigue dans son espace de problèmes, le programme reçoit une rétroaction c’est analogue aux récompenses, qu’il essaie de maximiser.

Pourquoi l’apprentissage automatique

  • L’apprentissage automatique est bon pour prévoir ou extraire des modèles (non linéaires) à partir des données.
  • L’apprentissage automatique n’est pas bon pour l’inférence statistique.

Types de tâches ML

Les tâches de ML courantes peuvent être divisées en quatre groupes :

  • Classification : Identifier à quelle catégorie appartient une observation.
  • Régression : Prédire une valeur continue.
  • Clustering : Groupement automatique d’observations.
  • Réduction de dimensionnalité : réduire le nombre de variables explicatives.

Un peu de vocabulaire

  • entraînement : estimation des paramètres et hyperparamètres du modèle
  • hyperparamètres : paramètres qui vont au-delà des poids des modèles (i.e. profondeur d’un arbre de décision)
  • réglage des hyperparamètres (tuning) : choisir les meilleurs
  • recherche sur grille (grid search) : essayer toutes les valeurs possibles sur une grille
  • prétraitement : toutes les étapes de préparation des données (normalisation, etc.)
  • score : fonction objective (i.e. MSE, R^2)

Estimer un modèle

  • Divisez l’échantillon en trois groupes : apprentissage, validation et test :
    • Mettez le jeu de test de côté jusqu’à la fin
    • Réglez les hyperparamètres :
      1. Estimez le modèle sur le jeu d’apprentissage
      2. Validez la performance OOS sur le jeu de validation.
      3. Optimisez les paramètres en répétant cette étape.
    • Alternative : validation croisée (mélanger apprentissage et validation)
    • Une fois satisfait, évaluez la vraie performance OOS sur le jeu de test.

Estimer un modèle de séries temporelles

  • En finance, nous nous soucions des propriétés des séries temporelles ;
    • L’optimisation peut être effectuée en utilisant des fenêtres roulantes.
    • Il est important que le jeu de test soit toujours hors échantillon.

Modèles de régression courants

  • Modèles linéaires (OLS, LASSO, Ridge, ElasticNet) :
    • {\hat {\beta }}\equiv {\underset {\beta }{\operatorname {argmin} }}(\|y-X\beta \|^{2}+\underbrace{\lambda_{2}\|\beta \|^{2}}_{\text{Ridge}}+\underbrace{\lambda_{1}\|\beta \|_{1}}_{\text{LASSO}})
    • Comprend également la régression logistique, la régression par quantiles, + beaucoup plus.
  • Machines à vecteurs de support (SVM): trouve les vecteurs (projections, peuvent être non linéaires) qui contiennent le plus de points de l’échantillon.
  • Arbres de décision : chaque branche est basée sur une décision binaire (i.e. x_1 > 0.04)

Méthodes d’ensemble

Nous voulons souvent combiner plus d’un modèle :

  • Si les erreurs ne sont pas parfaitement corrélées et que les modèles ne sont pas biaisés.
  • C’est la diversification !

Normalisation

Rappelez-vous, nous nous soucions uniquement de prédire y, pas d’inférence, donc il est courant de normaliser le X et y avant d’estimer le modèle pour améliorer les performances numériques.

  • Quelle que soit la transformation effectuée, elle doit être fixée avec le jeu d’apprentissage et la même transformation doit être appliquée aux observations hors échantillon.

Qu’est-ce que l’analyse de texte ?

  • L’analyse de texte, également appelée extraction de texte ou exploration de données textuelles, est le processus d’extraction d’informations de haute qualité à partir de texte.
  • L’analyse de texte implique la récupération d’informations, l’analyse lexicale pour étudier les distributions de fréquence des mots, la reconnaissance de motifs, le balisage/l’annotation, l’extraction d’informations, des techniques d’exploration de données y compris l’analyse des liens et des associations, la visualisation, et l’analytique prédictive.

Qu’est-ce que l’analyse de texte ? (suite)

  • L’objectif global est essentiellement de transformer le texte en données pour analyse, grâce à l’application du traitement du langage naturel (NLP), différents types d’algorithmes et de méthodes analytiques. Une phase importante de ce processus est l’interprétation des informations recueillies.

Pourquoi l’analyse de texte

  • Une quantité considérable d’informations précieuses est disponible au format texte, mais pas nécessairement dans un format structuré.
  • Les ensembles de données bien scrutcturés, tels que CRSP, Compustat, etc. ont été largement étudiés, donc la barre est haute pour répondre à des questions importantes en utilisant seulement ceux-ci.
  • Des questions intéressantes peuvent être répondues par ceux qui parviennent à trouver les bonnes données.

Les outils de l’analyse de texte

  • NLTK : Natural Language Toolkit. Puissante bibliothèque mettant en œuvre de nombreuses méthodes pour l’analyse de texte.
  • textblob : Package plus simple avec une interface plus facile à utiliser, construit sur NLTK et patterne.
  • Transformers : fournit des milliers de modèles pré-entraînés pour effectuer des tâches sur différentes modalités telles que le texte, la vision et l’audio.
  • spaCy : bibliothèque pour créer des pipelines NLP.
  • Voir également OpenAI, Ollama, Langchain listés plus tôt.
